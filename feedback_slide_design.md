When creating slides, follow McKinsey-style design principles as demonstrated in Reo's reference deck (Naigai long case).

**Why:** Reo is McK alumni and expects professional, insight-driven slides. Not decorative — every element must serve the argument.

**How to apply:**

## Core Principles

1. **Title = Message (Action Title)**
   - Every slide title is a complete sentence conveying the key takeaway
   - Good: "The market assigns negative value to the operating business as NFV / Market cap = 120%"
   - Bad: "Market Overview" or "Financial Analysis"

2. **Layout Patterns**
   - Most common: Left chart/data + Right bullets (insight/so what)
   - Tables with clear row/column structure
   - Waterfall charts for bridges
   - 2x2 positioning matrices for competitive landscape
   - ROIC tree decomposition for financial analysis
   - Timeline/phase tables for action plans

3. **Color & Design**
   - Dark navy blue (#003366) as primary
   - Light blue for secondary
   - Red for callout/emphasis only
   - Minimal palette — 2-3 colors max
   - Lots of white space
   - Clean, no decoration

4. **Structure**
   - Executive Summary as page 1 (entire argument in one page)
   - Agenda/divider pages between sections (dark blue bar highlights current section)
   - Page numbers bottom right
   - Source citations at bottom of every data slide
   - "Ref)" prefix for reference/backup slides

5. **Data Presentation**
   - Always include units, time period, source
   - Peer comparison for context
   - Time series for trends
   - Segment-level breakdown over consolidated-only

6. **Hierarchy within each slide**
   - Numbered improvement levers (①②③...)
   - Category labels (e.g., "Core Business", "Capital Policy", "Governance")
   - Tab-style navigation at top showing which topic area

7. **Engagement/Action Plan slides**
   - Phase-based timeline (0-3mo, 3-6mo, etc.)
   - Who / What / Agenda rows
   - Clear milestones
